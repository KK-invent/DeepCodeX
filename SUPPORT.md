# Support

## When to open an issue

- Something in the install or setup docs is wrong or confusing.
- A script fails and you can reproduce it.
- The doctor output doesn't make sense (even after you've redacted secrets).

## What to include

- Your macOS version
- Whether Codex.app and/or Deepcodex.app exist in `/Applications`
- The exact command that failed
- Redacted output from `deepcodex-doctor.py` or `scripts/preflight-mac.sh`

**Please don't paste** API keys, tokens, private URLs, or screenshots showing credentials. When in doubt, redact it.

## Security issues

Don't open a public issue — see [SECURITY.md](SECURITY.md) instead.
